## 1. 專案概述

### 目標
建立新湖地政事務所官方 LINE Bot，提供民眾地政業務諮詢服務，採純關鍵字自動回應機制，不使用 AI 動態回答。

### 服務範圍
- 登記業務諮詢（A 類）
- 測量業務諮詢（B 類）
- 地價業務諮詢（C 類）
- 資訊業務諮詢（D 類）
- 地用業務諮詢（E 類）
- 檔案應用及其他綜合業務諮詢（F 類）

### 核心運作邏輯
```
使用者傳送文字（手動輸入 或 點擊按鈕自動發送）
    ↓
Bot 比對關鍵字對照表
    ↓
找到對應 → 回傳指定訊息（文字 / 圖片 / 圖片輪播）
找不到   → 回傳預設引導訊息 + 六宮格圖片選單
```

## 4. 代碼對照表結構

所有內容統一管理於 `LINE_Bot_內容對照表.xlsx`，每筆資料包含以下欄位：

| 欄位 | 說明 | 必填 |
|---|---|---|
| 代碼 | 唯一識別碼（如 B1、REG-901） | ✅ |
| 功能說明（顯示文字） | 使用者看到的按鈕文字 | ✅ |
| 關鍵字（Key） | Bot 比對用的觸發字串 | ✅ |
| 回應文字1（W1） | 第一則回應文字 | ✅ |
| 回應圖片1（P1） | 第一則圖片 URL 或檔名 | ⬜ |
| 回應文字2（W2） | 第二則回應文字 | ⬜ |
| 回應圖片2（P2） | 第二則圖片 URL 或檔名 | ⬜ |
| 備註 | 開發備忘 | ⬜ |

## 5. 關鍵字觸發邏輯

### 觸發機制共四種

#### 機制一：Rich Menu 六宮格（底部固定）
使用者點擊底部圖示，LINE 自動發送對應文字或開啟連結。

| 代碼 | 關鍵字 / 動作 |
|---|---|
| RM-1 | 外部連結（官網） |
| RM-2 | 外部連結（案件查詢） |
| RM-3 | 外部連結（線上取號） |
| RM-4 | 外部連結（臉書） |
| RM-5 | 外部連結（預約） |
| RM-6 | LINE 分享功能 |

> Rich Menu 的連結類按鈕不需要 Webhook 處理，直接在 LINE OA Manager 設定即可。

#### 機制二：非關鍵字觸發
使用者輸入任何不在對照表內的文字，Bot 固定回傳三則訊息。

```javascript
// 判斷邏輯（index.js）
if (!keywordMap[text]) {
  // 回傳 NKW-1（文字）、NKW-2（文字）、NKW-3（圖片輪播）
}
```

#### 機制三：圖片輪播按鈕
點擊輪播圖片上的按鈕，LINE 自動幫使用者發送指定關鍵字（type: 'message' action）。

```
使用者點「鑑界規費怎麼算」按鈕
    → LINE 自動發送文字「鑑界規費試算」
    → Bot 收到「鑑界規費試算」
    → 查對照表找到 B1
    → 回傳 B1-W1 的內容
```

#### 機制四：數字指令
使用者直接輸入數字，Bot 回傳對應內容。

```
輸入「901」→ 查對照表 REG-901 → 回傳登記申請書下載連結
輸入「11」  → 查對照表 LAND-11 → 回傳非都市土地說明
```

### index.js 核心邏輯

```javascript
require('dotenv').config();
const express = require('express');
const line = require('@line/bot-sdk');
const { keywordMap } = require('./keywords');
const { getReply } = require('./menus');

const config = {
  channelSecret: process.env.CHANNEL_SECRET,
  channelAccessToken: process.env.CHANNEL_ACCESS_TOKEN
};

const client = new line.Client(config);
const app = express();

app.post('/webhook', line.middleware(config), (req, res) => {
  Promise.all(req.body.events.map(handleEvent))
    .then(() => res.sendStatus(200))
    .catch((err) => { console.error(err); res.sendStatus(500); });
});

async function handleEvent(event) {
  if (event.type !== 'message' || event.message.type !== 'text') return;

  const text = event.message.text.trim();
  const code = keywordMap[text];         // 查關鍵字 → 取得代碼

  if (code) {
    const messages = getReply(code);     // 查代碼 → 取得訊息陣列
    return client.replyMessage(event.replyToken, messages);
  }

  // 找不到關鍵字 → 回傳預設 NKW 組合
  const defaultMessages = getReply('NKW');
  return client.replyMessage(event.replyToken, defaultMessages);
}

app.listen(process.env.PORT, () => console.log(`Bot running on ${process.env.PORT}`));
```

### menus.js 結構

```javascript
// 代碼 → LINE 訊息物件陣列
// 每個函式回傳陣列，可包含 1~5 則訊息
module.exports.getReply = function(code) {
  const replies = {

    // 非關鍵字預設回應
    'NKW': [
      { type: 'text', text: 'XXX，您好！\n有任何地政相關問題，歡迎輸入以下數字...' },
      { type: 'text', text: '新湖地政官方帳號提供線上諮詢服務...' },
      carousels.mainMenu()   // 六宮格圖片輪播
    ],

    // B1 範例
    'B1': [
      { type: 'text', text: '【鑑界規費試算】\n...(填入說明文字)...' }
      // 若有圖片：{ type: 'image', originalContentUrl: 'https://...', previewImageUrl: 'https://...' }
    ],

    // A0 入口（回傳輪播）
    'A0': [
      carousels.registrationMenu1(),
      carousels.registrationMenu2()
    ],

    // ... 其餘代碼依對照表填入
  };

  return replies[code] || [{ type: 'text', text: '查無此服務，請輸入「主選單」回到首頁。' }];
};
```

## 8. 部署架構

### 選用方案：Render 免費方案

```
GitHub（程式碼）
    ↓ push 自動觸發部署
Render Web Service
    ↓ 提供 HTTPS 網址
LINE Webhook（https://your-app.onrender.com/webhook）
```

### 注意事項

- Render 免費版閒置 15 分鐘後會休眠，第一則訊息回應約慢 30 秒
- 解法：使用 [cron-job.org](https://cron-job.org) 免費服務，每 10 分鐘 GET 一次你的網址保持喚醒
- 喚醒端點建議加一個 `GET /` 回傳 200：
  ```javascript
  app.get('/', (req, res) => res.send('Bot is alive'));
  ```

### 與舊專案的關係

- 新專案與舊 Docker + CF Tunnel 專案**完全獨立**
- 新專案部署在 Render，使用 Render 提供的網址
- 舊 Docker 開不開不影響新 Bot 運作
- 兩者使用不同的 LINE Channel（不同 Token）

### 第1課：環境建置與第一個 Bot

**目標：** Bot 能收到任何訊息並回應「收到：xxx」

**步驟：**
1. 申請 LINE Developer 帳號
2. 建立 Messaging API Channel
3. 取得 Channel Secret 與 Access Token
4. 關閉 LINE OA 自動回應功能
5. 本機建立專案、安裝套件
6. 撰寫 index.js 基礎 Webhook
7. 部署至 Render
8. 設定 Webhook URL
9. 驗證：手機傳訊，Bot 回應「收到：xxx」

**完成條件：** Verify 成功，手機測試有回應

### 第2課：Rich Menu 六宮格

**目標：** 使用者開啟 Bot 時底部顯示六宮格選單

**步驟：**
1. 設計六宮格圖片（2500×1686px，分割為 2 行 3 列）
2. 在 LINE OA Manager 上傳圖片

**完成條件：** Bot 畫面底部出現六宮格，點擊可跳轉連結

## 10. AI Agent 協作指引

### 給 AI Agent 的說明

本文件是此專案的唯一真實來源（Single Source of Truth）。

在協作開發時請遵循以下規則：

1. **代碼命名不可更改**：所有代碼（A1、B3、REG-901 等）應與本文件及對照表完全一致

2. **關鍵字不可修改**：`keywords.js` 中的關鍵字必須與對照表「關鍵字（Key）」欄完全相符，包含全形半形、空格

3. **檔案職責分離**：
   - 路由邏輯 → `index.js`
   - 關鍵字對照 → `keywords.js`
   - 訊息內容 → `menus.js`
   - 輪播定義 → `carousels.js`

4. **訊息格式規範**：
   - 純文字 → `{ type: 'text', text: '...' }`
   - 圖片 → `{ type: 'image', originalContentUrl: '...', previewImageUrl: '...' }`
   - 輪播 → Flex Message carousel 格式
   - 每次回覆最多 5 則訊息

5. **開發優先順序**：
   - 先讓架構跑通（第1課）
   - 再補內容（第5課後）
   - 最後補圖片（第10課）

6. **提問格式建議**：
   ```
   「請幫我完成代碼 B1，
     關鍵字：鑑界規費試算，
     W1：[此處貼上你的說明文字]，
     無圖片。」
   ```

建議在對照表中增加「狀態」欄：

| 狀態值 | 說明 |
|---|---|
| 待填 | 內容尚未確定 |
| 已填 | 內容已填入對照表 |
| 已實作 | 已寫入程式碼 |
| 已測試 | 通過手機測試 |
